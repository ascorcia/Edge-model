# DEVELOPMENT PROTOCOL

## Strict QA-First Development Protocol
1. All code changes must pass QA review before merging.
2. Code must adhere to the defined coding standards and guidelines.

## Pre-Delivery Testing Steps
1. Run unit tests on all new code.
2. Perform code reviews peer-reviewed by at least two team members.

## Data Flow Verification Procedures
1. Validate data input sources for accuracy.
2. Implement monitoring to track data flow through applications.

## Cross-Feature Integration Testing
1. Ensure that new features do not interfere with existing functionalities.
2. Run integration tests that cover all relevant feature combinations.

## Feature Validation Checklist Workflow
1. Create a checklist for each feature.
2. Review and confirm completion of all checklist items before feature delivery.
